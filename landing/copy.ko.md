## 3. For Agent Builders（Agent 개발자）

### 몇 분 만에 어떤 API든 연결하세요, 며칠이 아니라

원스톱 통합을 더 이상 작성하지 마세요. AgenticTrade의 MCP Descriptor를 통해 Agent가 네이티브 워크플로우에서 등록된 모든 API를 발견하고 연결할 수 있습니다.

**작동 원리：**

1. Agent가 외부 API가 필요한 작업을 만나면
2. MCP Tool Descriptor를 통해 AgenticTrade 레지스트리에 쿼리를 날립니다
3. Agent는 엔드포인트 스펙, 인증 방식, 프록시 키를 수신합니다
4. Agent가 API를 호출하고 USDC/USDT로 자동 결제합니다
5. 데이터는 귀하에게. 코드에 결제 로직이 필요 없습니다.

**왜 중요한가：**

- Agent가 신용카드 처리할 필요 없음
- 결제용 Webhook 핸들러 구축 불필요
- API Key 노출 걱정 불필요
- Agent가 머신 스피드로 마이크로 트랜잭션 결제

## 4. How It Works（운영 프로세스）

### API 제공자용（API 등록）

```
1단계 → 가입 → 2단계 → API 스펙 및 가격 제출 → 3단계 → MCP Descriptor 승인 → 4단계 → 라이브 개시
```

- **1단계：** 제공자 계정 생성（이메일 또는 암호화폐 지갑）
- **2단계：** API 엔드포인트 스펙, 속도 제한, 가격 정책 제출
- **3단계：** AgenticTrade가 스펙에서 자동으로 MCP Tool Descriptor 생성
- **4단계：** API 라이브 개시. Agent가 24/7 자동으로 발견하고 호출합니다.

### Agent 개발자용（API 사용）

```
1단계 → Agent가 API 필요 → 2단계 → 레지스트리 쿼리 → 3단계 → 스펙 및 프록시 키 수신 → 4단계 → 호출 및 결제
```

- **1단계：** Agent가 외부 데이터나 기능이 필요한 작업을 수행합니다
- **2단계：** Agent가 기능 쿼리로 AgenticTrade MCP 레지스트리를 호출합니다
- **3단계：** 레지스트리가 스펙과 프록시 액세스 키가 포함된 해당 API를 반환합니다
- **4단계：** Agent가 직접 API를 호출합니다; 정산은 자동 및 즉각적입니다

## 7. FAQ

### API 제공자용

**Q: 제 API에는 인증이 필요합니다. 어떻게 작동하나요?**
A: 등록할 때 인증 방식을 정의하세요（API Key, OAuth, JWT）. 실제 자격 증명에 매핑되는 프록시 키를 생성해 드립니다. Agent는 실제 키를 절대 볼 수 없습니다.

**Q: Agent가 너무 많이 호출하면 어떻게 하나요?**
A: 각 엔드포인트별 속도 제한을 설정할 수 있습니다. Agent는 이를 준수해야 하며, 그렇지 않으면 스로틀링됩니다. 상한선을 귀하가 설정합니다.

**Q: 등록을 취소할 수 있나요?**
A: 네. 언제든 가능합니다. 기존 프록시 키는 몇 분 내에 취소됩니다. 계약 갇힘 없음.

**Q: 어떻게 결제받나요?**
A: USDC 또는 USDT를 지갑으로 또는 은행으로 텔레그라피 송금. 정산은 주간 또는 요청 시.

---

### Agent 개발자용

**Q: 제 Agent는 어떻게 API를 발견하나요?**
A: MCP Tool Descriptor 프로토콜을 통해. Agent가 기능 요청으로 레지스트리에 쿼리하면, 해당 API가 자동으로 스펙을 반환합니다.

**Q: API가 다운되면 어떻게 하나요?**
A: Agent는 재시도 로직으로 실패를 우아하게 처리합니다. 업타임 통계를 제공하여 Agent가 신뢰할 수 없는 엔드포인트를 피할 수 있습니다.

**Q: 암호화폐 대신 법화로 결제할 수 있나요?**
A: 현재 USDC/USDT를 지원합니다. Agent의 법화 결제는 로드맵에 있습니다（2026년 4분기）.

**Q: Agent용 무료 플랜이 있나요?**
A: 네. 각 Agent당 매월 첫 $10 어치 API 호출은 무료입니다. 신용카드 필요 없음.
