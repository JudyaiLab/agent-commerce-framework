## FAQ

**Q: What makes AgenticTrade different from RapidAPI?**
A: Two things. First, agents can pay here — not just humans. Your AI agent can have its own wallet, discover APIs, and pay for usage without you touching anything. Second, our commission is 0% → 5% → 10% vs RapidAPI's flat 25%. For a provider doing $10K/month, that's $2,500/month in your pocket instead of theirs.

**Q: What is an MCP Tool Descriptor?**
A: MCP (Model Context Protocol) Tool Descriptor is a machine-readable description of what an API does, what parameters it accepts, and what it returns. Your agent reads it, understands how to call the API, and calls it correctly — without you hardcoding anything.

**Q: How does the Proxy Key system work?**
A: When you list an API, you give us a proxy of your real key — not the key itself. Agents use the Proxy Key to call your API through our infrastructure. Your real key is never exposed. You can rotate it anytime without touching agent code.

**Q: What tokens does AgenticTrade accept?**
A: Providers can choose to receive in USDC, USDT, or fiat (via PayPal). Agent builders fund their wallet in USDC or USDT.

**Q: Can I use AgenticTrade if my agent isn't a crypto-native application?**
A: Yes. Agent builders don't need to know anything about crypto. You fund a wallet in USDC using a card, your agent handles the rest. For providers, you can receive fiat and never touch a wallet.

**Q: How does an agent discover my API?**
A: The MCP registry is searchable by capability. An agent can query it like: "find me a weather API for Seoul that returns temperature in Celsius." If your MCP Descriptor matches, the agent knows how to call you, what it costs, and can pay in one step.

**Q: What happens if my API goes down?**
A: AgenticTrade doesn't control your API — you're the provider. We handle payment failures and retries, but availability is between you and the agents consuming your service. SLA commitments are between you and your users.
